# SpicyPrompter

Batch AI prompt generator for image workflows. Connect any OpenAI-compatible LLM, randomize categories, generate hundreds of prompts, and pipe them straight into ComfyUI.

## Features

- **Batch generation** — unlimited prompts per run *(paid)*; up to 250 total in demo (5 per run)
- **Category system** — acts, scenes, themes, styles, roles, body types, races — each with curated subject/tag pools
- **Random mode** — per-slot randomization across all categories; chosen value shown in log per prompt
- **ComfyUI integration** — sends each prompt directly to your ComfyUI instance via API workflow
- **LLM agnostic** — any OpenAI-compatible endpoint (Ollama, llama.cpp, LM Studio, OpenRouter, etc.)
- **Prompt adaptation** — optional second LLM pass rewrites structured prompts into natural prose or comma-separated tags to match your workflow style
- **LoRA Dataset Builder** — dedicated tab for generating portrait prompt sets optimized for LoRA training, with ZIP export
- **Cast enforcement** — solo/all-female subjects automatically block male-requiring acts; random pools filter to compatible acts only
- **Trigger words** — per-workflow trigger words auto-detected from LoRA metadata, prepended to every generation
- **Hair color / Eye color / Skin tone selectors** — optional selectors in the Body tab with ABSOLUTE LLM enforcement
- **Camera view tab** — top-down, bottom-up, side, eye-level, wide shot, dutch angle; LLM-enforced viewpoint
- **Facial expression selector** — choose from a curated list; expression is enforced in generated prompts
- **A1111 import** — import PNG images generated by Automatic1111; extracts prompt and generation params as read-only reference
- **Inline prompt editing** — edit and save positive prompts directly on the detail page
- **Original** — regenerate using a prompt's embedded workflow, bypassing global settings
- **Sampling controls** — temperature, top-p, min-p, repetition penalty, max tokens
- **License gating** — demo mode with restricted category pool; full access with license key

## Requirements

- Windows 10/11
- Node.js 18+ (or none — installer downloads portable Node automatically)
- A running OpenAI-compatible LLM endpoint
- ComfyUI (optional, for image generation)

---

## Install

1. Download from [Releases](https://github.com/magnetik713/spicyprompter/releases)
2. Extract anywhere
3. Run `install.bat` — installs dependencies, prompts for port (default 3016)
4. Browser opens to `http://localhost:3016/prompts`

After first install, use `start.bat` to launch.

## Setup

Go to **Settings** and configure:

### LLM Inference

| Setting | Description |
|---|---|
| Base URL | Your LLM endpoint, e.g. `http://localhost:11434/v1` |
| API Key | Leave blank for local, or paste your key |
| Default Model | Model name, e.g. `qwen3.6:35b-a3b` |

### Generation Parameters

| Setting | Default | Notes |
|---|---|---|
| Temperature | 1.2 | Higher = more creative |
| Top-p | off | Nucleus sampling cutoff |
| Min-p | off | Cuts tokens below fraction of top token probability. Try 0.05-0.1 with high temp |
| Repetition Penalty | 1.1 | Discourages repeated phrasing. Ollama only |
| Max Tokens | 300 | Per prompt |
| Raw output | off | Strips LLM preamble/postamble |

### Prompt Adaptation

Optional — rewrites the generated prompt to match your workflow's expected style before sending to ComfyUI.

| Setting | Description |
|---|---|
| Base URL | Adaptation LLM endpoint — can differ from generation LLM |
| API Key | Leave blank for local |
| Model | Model to use for adaptation |

Workflow type is auto-detected (SDXL, SD 1.5, FLUX/krea2) and the rewrite style adjusts accordingly.

### ComfyUI

| Setting | Description |
|---|---|
| Host | ComfyUI host, e.g. `192.168.1.30` |
| Port | ComfyUI port, e.g. `8188` |
| Workflow File | Upload an API-format workflow JSON |
| Trigger Words | Auto-populated from LoRA metadata. Prepended to every generation |
| Model | Checkpoint filename to inject into workflow |
| Positive Node ID | Leave blank — auto-detects. Set only if auto-detect picks wrong node |
| Negative Node ID | Leave blank — auto-detects |
| Prompt Suffix | Appended to every positive prompt |

#### ComfyUI Overrides

| Setting | Notes |
|---|---|
| Steps | KSampler steps |
| CFG | CFG scale |
| Denoise | 0.0-1.0 |
| Guidance | FluxGuidance node value |
| Sampler | Must match ComfyUI sampler names exactly (e.g. `euler`, `dpmpp_2m`) |

## Generate Tab

- **Role / Body Type / Race** — filter subject pool
- **Act / Scene / Theme / Style** — set category context; pick a specific value or **Random**
- **Count** — number of prompts to generate
- **Send to ComfyUI** — queue each prompt as it completes

## Dataset Builder Tab

Generates portrait prompt sets optimized for LoRA character training.

**How it works:**

1. Choose subject attributes (race, body type, age, hair, clothing, etc.)
2. Select camera views — Generate cycles through each automatically
3. Click **Generate** — prompts are written to the library tagged by view
4. Click **Download ZIP** — exports the full set as a ready-to-use training dataset

**ZIP contents:**

| File | Description |
|---|---|
| `001_portrait.txt`, `002_full_body.txt`, ... | One prompt per camera view, named by angle |
| `captions.csv` | All captions in CSV format for tools that prefer it |
| `reference.jpg` | Optional face photo — attach to IP-Adapter / PuLID / InstantID in ComfyUI for face consistency |
| `README.txt` | Quick reference |

**Using the dataset:**

Run each prompt in ComfyUI and save images with matching filenames (`001_portrait.png` pairs with `001_portrait.txt`). Feed image + caption pairs into your training tool (Kohya-ss, SimpleTuner, OneTrainer). 20-40 images is a good target; the default 7 views x 1 per view gives a solid starting set.

---

## Detail Page

- Click the positive prompt text to edit it directly; hit **Save** to persist
- **Regenerate** — queues the prompt using your current Settings workflow
- **Original** — regenerates using the workflow embedded in the image at time of generation
- Override fields (Steps, CFG, Sampler, Denoise, Seed) — per-regeneration overrides

---

## License

Demo mode is active without a key — a curated subset of categories is available, with generation capped at 5 prompts per run (250 total). Enter a license key in Settings to unlock all categories and remove generation limits.

## Uninstall

Run `uninstall.bat` or delete the folder. No registry entries, no system-wide installs (if portable Node was used).
